# AeroFlow

## Система управления аэропортовыми оповещениями

AeroFlow — платформа для управления аэропортовыми объявлениями, расписанием рейсов и процессами работы диспетчера.

Система является развитием существующей САО и должна сохранить совместимость с текущей логикой работы: диспетчер управляет рейсами и объявлениями, а система автоматически ставит объявления в очередь и воспроизводит их по правилам приоритета.

---

# Репозитории

```text
aeroflow-core        # Symfony backend, бизнес-логика
aeroflow-web         # React + Mantine, интерфейс администратора и диспетчера
aeroflow-playback    # сервис очереди и управления воспроизведением
aeroflow-agent       # локальный аудио-агент
```

---

# Основные роли

## Администратор

* управление пользователями;
* управление рейсами;
* управление расписанием;
* настройка шаблонов объявлений;
* загрузка аудиофайлов;
* настройка языков;
* просмотр истории действий.

## Диспетчер

* открыть регистрацию;
* закрыть регистрацию;
* начать посадку;
* объявить прибытие;
* запустить дополнительное объявление;
* запустить экстренное объявление;
* контролировать очередь воспроизведения;
* остановить текущее объявление.

# Архитектура

## aeroflow-core

Основной бизнес-сервис.

Отвечает за:

* рейсы;
* расписание;
* статусы рейсов;
* объявления;
* шаблоны объявлений;
* аудиофайлы;
* языки;
* пользователей;
* аудит.

`aeroflow-core` не воспроизводит звук напрямую.

Он создаёт команды на воспроизведение и отправляет их в `aeroflow-playback`.

---

## aeroflow-playback

Сервис управления очередью воспроизведения.

Отвечает за:

* получение команд на воспроизведение;
* постановку объявлений в очередь;
* управление приоритетами;
* запуск текущего объявления;
* остановку текущего объявления;
* отмену будущих объявлений;
* автоматические повторы;
* экстренный режим;
* хранение состояния очереди.

`aeroflow-playback` не знает бизнес-логику рейсов.

Для него существуют только задания воспроизведения.

---

## aeroflow-agent

Локальный агент воспроизведения.

Отвечает за:

* получение команды `play`;
* воспроизведение аудиофайлов;
* остановку воспроизведения;
* паузу;
* продолжение;
* управление громкостью;
* отправку статуса обратно в `aeroflow-playback`.

Агент не содержит бизнес-логики.

---

## aeroflow-web

Веб-интерфейс системы.

Содержит:

* интерфейс администратора;
* интерфейс диспетчера;
* экран расписания;
* экран очереди воспроизведения;
* экран управления аудиофайлами.

---

# Взаимодействие компонентов

```text
aeroflow-web
     ↓
aeroflow-core
     ↓
RabbitMQ
     ↓
aeroflow-playback
     ↓
RabbitMQ / WebSocket / HTTP
     ↓
aeroflow-agent
```

---

# Текущий ручной сценарий

```text
Диспетчер выбирает FlightDefinition
  ↓
Нажимает действие объявления
  ↓
Система создаёт Announcement
  ↓
Announcement отправляется в playback
```

`FlightDefinition` является постоянной карточкой и не создаётся для каждого
фактического вылета.

Справочником `FlightDefinition` управляет администратор. Диспетчер использует
активные карточки в отдельном рабочем интерфейсе запуска объявлений.

В будущем могут быть добавлены:

* `FlightSchedule` — правило расписания;
* `FlightOccurrence` — конкретный вылет и его операционное состояние.

Они не требуются для первого ручного режима.

# Типы объявлений

## Начало регистрации

Воспроизводится один раз.

После запуска система переводит рейс в состояние `Регистрация открыта`.

## Продолжение регистрации

Создаётся автоматически через заданный интервал, пока регистрация открыта.

## Окончание регистрации

Останавливает автоматические повторы объявления о продолжении регистрации.

## Приглашение на посадку

Сообщает о начале посадки и номере выхода.

## Прибытие рейса

Сообщает о прибытии рейса.

## Дополнительное объявление

Информационное сообщение аэропорта.

## Экстренное объявление

Объявление максимального приоритета.

При запуске экстренного объявления обычная очередь приостанавливается.

---

# Приоритеты воспроизведения

```text
1000  Экстренное объявление
100   Рейсовое объявление
50    Дополнительное объявление
0     Фоновая музыка
```

---

# Очередь воспроизведения

Каждое объявление превращается в задание воспроизведения.

```text
PlaybackJob
- id
- type
- priority
- status
- scheduledAt
- audioSequence
- repeatRule
- createdAt
```

Статусы:

```text
Pending
Scheduled
Playing
Completed
Interrupted
Cancelled
Failed
```

---

# Поддержка старой САО

Новая система должна уметь использовать существующие аудиофайлы.

Пример структуры:

```text
Звуки
├── Вылеты
├── Прибытие
├── Дополнительно
├── Экстренные
├── Числа
├── Гонги
└── Музыка
```

Это позволит выполнить постепенный переход без полной перезаписи аудиоконтента.

---

# Режимы работы

## Ручной режим

Диспетчер вручную запускает объявления, как в старой системе.

## Полуавтоматический режим

Система подсказывает действия на основании расписания.

## Автоматический режим

Система сама создаёт объявления на основании расписания и состояния рейса.
