# Flight Definition Fixtures

Status: Done  
Bounded context: Flight Operations  
Service: aeroflow-core  
Change type: Infrastructure, Development Data

## Goal

Добавить идемпотентную fixture-группу с демонстрационными рейсами международного
аэропорта Кишинёва для локальной разработки административного интерфейса и
будущих диспетчерских сценариев.

## Decisions

* используется актуальный IATA-код аэропорта Кишинёва `RMO`;
* данные представляют типичные регулярные направления, но не являются
  операционным расписанием;
* для каждого направления добавлены вылет и прибытие;
* существующие карточки с тем же бизнес-ключом пропускаются;
* domain events, созданные агрегатом при построении fixture, очищаются и не
  публикуются;
* fixture доступна в группе `flight-operations`.

## Usage

```text
php bin/console doctrine:fixtures:load --group=flight-operations --append
```

## Test Scenarios

* создаются 12 вылетов и 12 прибытий;
* для вылетов `RMO` является аэропортом отправления;
* для прибытий `RMO` является аэропортом назначения;
* повторная загрузка пропускает существующие бизнес-ключи;
* fixture не оставляет domain events для публикации.
