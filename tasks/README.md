# AeroFlow Tasks

Эта папка содержит рабочие спецификации вертикальных срезов.

Task отвечает на вопрос:

```text
Что именно реализуется сейчас и как проверить результат?
```

Долгоживущие правила после завершения задачи должны быть отражены в:

* `domain-model.md` — доменная модель и инварианты;
* `architecture.md` — границы компонентов и интеграции;
* `README.md` — обзор продукта.

## Статусы

* `Draft` — спецификация уточняется, реализация не начата;
* `Ready` — решения приняты, задачу можно реализовывать;
* `In Progress` — реализация начата;
* `Done` — критерии готовности выполнены;
* `Superseded` — задача заменена другой спецификацией.

## Именование

```text
NNN-short-task-name.md
```

Одна задача должна давать проверяемый вертикальный результат и не объединять
несколько bounded contexts без необходимости.
